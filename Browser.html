What is Web Browser?

A web browser takes you anywhere on the internet. It retrieves information from other parts of the web and displays it on your desktop or mobile device. The information is transferred using the Hypertext Transfer Protocol, which defines how text, images and video are transmitted on the web.

Web Browser interface

  • TAB bar: Where the box is spawned when the user opens a new website or new Tab.

  • Address Bar: In the address bar, user type in the website domain where the users want to go. When the user don't types the domain, it will reponse as searching box like search engine and redirects to google.

  • Bookmarks: Bookmarks or bookmark bar is where the user save what ever their website they want. User can set the name of the bookmark. Most of the browsers also have bookmark lists where users can save their websites but don't want to show it on the main page.

  • Main page: Visual of the main page depends on what setting and what browser do the user use. In case of Firefox Developer Edition, there is a blue Firefox logo with google search bar and most often visited websites with recent activity shown.

The Workers Browser Rendering API allows developers to programmatically control and interact with a headless browser instance and create automation flows for their applications and products. Once you configure the service, Workers Browser Rendering gives you access to a WebSocket endpoint that speaks the DevTools Protocol.

Compatibility of Browsers

  • Different Browsers: There are several browser plateforms like Google Chrome, Mozilla Firefox, Safari, Opera and etc. If people use different browsers and that browsers support different protocols, then people won't see the same page in different browsers.

  • User Experience: Problem that is directly connected to different browsers. When people look at different screen, users will get different user experience. For example, when user go to youtube.com, in google chrome, it works well. However, if the user used different browser that doesn't support the protocol that youtube use, then user could have different screen.
