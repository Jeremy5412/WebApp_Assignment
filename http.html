HTTP

Hypertext Transfer Protocol is an Application layer( a way of hiding specific working data that specifies the shared communication protocols and interface methods used by hosts in a communication network. ) protocol for transmitting hypermedia documents, such as HTML, XML.

Designed for communication between web browsers and web servers. It follows classical client-server model(the way of communication that we think what device and servers are doing. ) It opens a connection to send the request and wait for the request to come from the server.

It is a stateless protocol. Server don’t need to storage the data that have requested from the client.

HTTP requests are how clients (like web browsers) talk to servers on the internet. Clients send requests with information like what they want and where to get it. Servers process these requests and send back HTTP responses. Responses include a status code, headers, and the actual content. This process is fundamental to how the internet works, allowing users to access web pages, retrieve data, and interact with online resources.

Web browsers make websites visible to users by interpreting HTML files, which consist of HTML elements composed of tags and attributes. These elements serve as the foundational building blocks of web pages. Some HTML elements for structing contents are Header, Body, and footer tag.

There are over 50 unique HTTP status codes across five different categories. And these are three status code that users often see.

  • 404 Not Found: This could be used for example if you try to access a URL that does not exist or if you try to access something from the database that does not exist.
  • 401 Unauthorized: while it uses the word unauthorized (which means you don’t have permission) this status code actually means you are unauthenticated.
  • 403 Forbidden: This status informs the client that they do not have permission to perform this request.

When a URL is entered into the browser, the browser sends an HTTP command to the web server directing it to search and transmit the requested web page. For example, when I go into naver.com, client requests the information for the webpage to display on their screen and the server respone to that.

  6. Critical Thinking: Understanding web structures and HTTP empowers users to critically assess online information, an essential skill for digital citizenship.
